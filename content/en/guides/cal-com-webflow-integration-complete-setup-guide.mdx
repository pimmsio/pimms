**Cal.com + Webflow integration** lets you embed Cal.com directly into your Webflow site with complete booking attribution. The Pimms dashboard includes interactive setup wizards for both Cal.com and Webflow — two quick wizards and you're tracking meetings end-to-end.

This is the ideal setup for agencies, consultants, and service businesses who use Webflow to build their marketing site and Cal.com to schedule client calls. You'll see exactly which campaigns generate actual booked meetings through your Webflow pages.

Webflow is great for building professional marketing sites with embedded booking widgets. Pimms adds the missing piece: **attribution**. Now you know which page, campaign, or ad brought each prospect who booked a meeting.

## Why track Cal.com bookings on your Webflow site?

- **Know which service page generates the most consultations.** You have separate Webflow pages for each service you offer, each with an embedded Cal.com booking widget. Pimms shows you which pages drive the most bookings and from which traffic source.

- **Prove ROI to clients.** If you're an agency running campaigns that drive traffic to a Webflow site with Cal.com booking, Pimms gives you clear data to show clients: "Your LinkedIn campaign generated 14 demo bookings this month."

- **Compare SEO vs paid traffic for meeting generation.** Your Webflow site ranks organically and you also run paid ads. Both drive traffic to your booking page. Pimms shows you which source generates more meetings — so you know where to invest.

## Prerequisites

## What you'll track

- **Cal.com bookings from your Webflow site** — Every meeting booked through your embedded Cal.com widget
- **Meeting attribution by campaign** — Which ad, email, social post, or referral source drove each booking
- **Page-level performance** — See which Webflow pages generate the most meetings
- **Channel comparison** — Compare how different traffic sources perform at generating bookings

## How to set it up

This integration combines two setup wizards in your [Pimms dashboard](https://app.pimms.io):

1. **Complete the Cal.com wizard first** — This configures the webhook that sends booking events to Pimms and adds the tracking field to your Cal.com forms
2. **Then complete the Webflow wizard** — This adds the detection script to your Webflow site so visitors from Pimms links are identified

Both wizards are guided step-by-step. Total setup time is about 10 minutes for both.

## What the setup wizards cover

**Cal.com wizard (6 steps):**
1. Install the detection script on your Webflow site
2. Verify script installation
3. Add the pimms_id tracking field to Cal.com booking questions
4. Configure the Cal.com webhook
5. Create a test link
6. Verify tracking works

**Webflow wizard (6 steps):**
1. Install tracking scripts (detection + form injection)
2. Verify script installation
3. Build a Pimms-compatible form (if applicable)
4. Configure the Webflow webhook (for form tracking)
5. Create a test link
6. Verify tracking works

If you only need Cal.com booking tracking (no Webflow forms), the Cal.com wizard alone is sufficient.

## How it works

1. **The Pimms detection script** is installed on your Webflow site — when a visitor arrives via a Pimms link, the script detects the click ID
2. **The script auto-injects the tracking ID** into Cal.com embeds on the page via the hidden `pimms_id` field
3. **The visitor books a meeting** through your embedded Cal.com widget — the booking includes the Pimms click ID
4. **Cal.com fires a webhook** to Pimms with the booking data and tracking ID
5. **Pimms attributes the meeting** to the original campaign, visible in your dashboard

The detection script handles the handoff between Webflow and Cal.com automatically — no manual URL parameter passing needed.

## FAQs

::: faq

### Do I need a paid Webflow plan for Cal.com integration?

It depends on your setup. Webflow's free plan allows some custom code, but a paid plan gives full access to head code injection and site-wide scripts. Most users have a paid Webflow plan and can add the tracking script easily.

:::

::: faq

### Can I customize the Cal.com booking form appearance?

Yes! Cal.com offers extensive customization options for colors, branding, and form fields. You can customize these in your Cal.com dashboard under Event Types → Advanced → Booking Questions.

:::

::: faq

### Will this work with Webflow's CMS?

Yes! You can use Webflow's CMS to create dynamic booking pages with different Cal.com links for different team members or services. The detection script works across all pages, including CMS collection pages.

:::

::: faq

### Can I use Pimms short links for the fullscreen calendar method instead of adding ?pimms_id=1?

Yes, but this approach breaks attribution tracking. When you use a Pimms short link directly to Cal.com, all bookings will be attributed to the button click rather than the actual traffic source.

**Use Pimms short links only for:**
- Short-term A/B testing (max 1 month)
- Testing which button design converts better
- Quick setup when you need immediate tracking

For long-term tracking and proper attribution, always use the Cal.com link with `?pimms_id=1` parameter.

:::

::: faq

### Can I track both Webflow form submissions AND Cal.com bookings?

Yes! If you have both Webflow forms and Cal.com booking widgets on your site, setting up both wizards will track all conversions. Form submissions and meeting bookings will both appear in your Pimms dashboard with full attribution.

:::
